Architecture (§19.2, §19.3)

Two coordinated interfaces over one deterministic core. The LLM plane holds a read + Draft-only credential and no database access — chat never owns financial logic, approval authority, or execution state.

Deterministic plane

Go owns all money, identity, event, policy, approval, execution, reconciliation, audit, and scheduling. Monorepo: go, python, web, extension, contracts. Go OpenAPI is the contract source; Python/TS clients are generated with a CI drift check.

Model plane rules

Model/provider selection is configuration; Gate 0a picks the cheapest pair clearing every threshold. If the provider degrades, chat falls back to suggested prompts + screens — never to an unqualified model. Failure: one automatic retry, then deep-link.

Deploy & observe

Docker Compose on one production VPS + isolated backup, Caddy ingress, OpenTelemetry with Grafana/Loki/Tempo. Route C: Go HTTP mainline; chromedp only if Gate 0 proves rendering necessary — else the wedge does not launch.

Scope ladder

P0 proves one connector, one locale, one region, one model configuration. Generalization waits for a second marketplace to demonstrate it.

    Approval state machine (§8.4)

    One action path for chat and screens. Free text never approves — only a structured control bound to action ID, parameter version, context version, and expiry. Unknown external result is neither success nor failure: it stays Pending Reconciliation until current state is read.

    Rollback is a new recommendation + approval with its own action ID — never an automatic inverse write (EXE-004). Duplicate confirmations reject; retry is blocked while any item is unreconciled.

    Money, contribution & policy (§9)

    Money representation

    Money { mantissa: int64, currency: ISO-4217, exponent: int8 }
    • No floating point ever enters a money path; rates are fixed-point basis points
    • Arithmetic only via Money methods; mixed currency/exponent rejects
    • Raw marketplace text/value/unit preserved separately as evidence
    • Ambiguous source unit → quarantine, never inference
    • No cross-currency conversion in P0; static lint bans raw integer math in money/margin/policy/card packages
    • Toman display only through the verified, versioned IR region transform; inexact transforms also show the exact source-unit amount

    Policy order — later rules cannot override earlier ones

    Default movement cap 5%, cooldown 60 min — P0 accounts may only tighten. No action may cross zero contribution.

    Contribution model

    net proceeds − COGS − commission − fulfillment − seller-funded shipping − packaging − seller-funded promotion − variable ads − expected returns. Only Complete readiness can drive an executable recommendation; Partial shows analysis only; Stale/Missing block.

    Conversational layer (§8, §12)

    Chat is not a seventh product area. One active context per conversation, visible as a chip; ambiguity always produces a picker before any card exists. If chat dies, every P0 workflow still works on screens.

    Administration levels

    Intent classes & containment

    • Only Prepare Action can create a Draft; no model tool can approve, execute, confirm results, or write guardrails/permissions (registry-tested)
    • Model cannot calculate authoritative price/contribution or override any gate
    • Every numeric financial value comes from engine outputs; responses fail closed on missing evidence
    • Eval set before beta: 630 cases · ≥90% intent, ≥95% context, 100% adversarial approval containment, ≥95% factual support

    Success model (§5)

    North star — WVRA: weekly value-realizing accounts completing ≥ 1 platform-recommended, margin-safe action on Confirmed identity, Complete cost, fresh evidence. Write mode needs a reconciled terminal result; recommend-only needs a matching owned-price observation within 24 h. Message count is an anti-metric.

    Localization boundary (§11)

    Architecture

    • One UI locale (fa-IR) and one region (IR) over a locale-neutral core — no locale/calendar/currency/direction branch in core logic (LOC-001)
    • Versioned locale pack: catalog, direction, digit families, number format, calendar, collation, plurals, prompt/eval assets
    • Region config: currency, verified source-unit mapping, display transform, timezone, business days, connector binding
    • Persian/Latin digits normalize before calculation; Jalali is a display calendar over UTC storage; Latin identifiers are bidi-isolated
    • Pseudo-locale + copy-lint are CI gates; missing keys fall back to English + telemetry
    • A new locale/region must require only pack + config + connector binding — zero core diff (LOC-010)

    Canonical state glossary (fa-IR) — the single copy source

    Release gates (§20)

    Internal alpha

    • All P0 requirement tests pass; Gate 0 thresholds stay green
    • Screens-only kill-switch journey passes
    • Adversarial approval suite contains 100%
    • RTL/bidi, Jalali, pseudo-locale, pseudo-currency, fallback pass
    • No Unknown capability enables a control
    • Runbooks + analytics match source records

    Private beta

    • 5 production accounts connected
    • ≥ 70% of each pilot assortment Complete & observation-ready
    • One-week priority-freshness soak passes
    • Writes verified or account visibly recommend-only
    • Briefing ≥ 95% during soak; P75 economics support tested price

    Paid beta / GA

    • WVRA ≥ 3 of 5 for four consecutive weeks
    • Precision, freshness, sync, grounding, context, safety targets pass
    • Zero unresolved sev-1 identity/money/policy/execution defects
    • ≥ 70% gross margin at P75; ≥ 3 partners accept paid price
    • Support load sustainable without hidden concierge work

    Autopilot stays off (P1 proof gate, §19.4)

    ≥ 500 approved human actions · zero guardrail breaches · ≥ 95% reconciled success · ≥ 90% policy agreement on a labeled set · explicit account + SKU-group opt-in · independent pause control. Chat may report and pause autopilot; it can never enable it.
